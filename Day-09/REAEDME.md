# Day 9

## Problem 28

* **Concept:** Last Positive Number
* **Time Complexity:** O(n)
* **Space Complexity:** O(1)
* **Lesson Learned:** Last occurrence problems are solved by updating the answer whenever the condition is satisfied instead of returning immediately.

## Problem 29

* **Concept:** Last Even Number
* **Time Complexity:** O(n)
* **Space Complexity:** O(1)
* **Lesson Learned:** The last occurrence pattern can be reused by changing only the filtering condition.

## Problem 30

* **Concept:** Last Index of Target
* **Time Complexity:** O(n)
* **Space Complexity:** O(1)
* **Lesson Learned:** The same last occurrence pattern works for indices as well as values by tracking the latest matching index during traversal.

## Bonus Learning

* **Concept:** Last Occurrence Pattern
* **Lesson Learned:** Unlike first occurrence problems that return immediately, last occurrence problems require traversing the entire list and overwriting the stored answer whenever a valid element is found.
