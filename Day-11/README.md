# Day 11

## Problem 36

* **Concept:** Count Positive Even Numbers
* **Time Complexity:** O(n)
* **Space Complexity:** O(1)
* **Lesson Learned:** Counting problems can be combined with multiple filtering conditions using logical operators.

## Problem 37

* **Concept:** Sum of Negative Odd Numbers
* **Time Complexity:** O(n)
* **Space Complexity:** O(1)
* **Lesson Learned:** Summation problems follow the same traversal pattern as counting problems; only the accumulator operation changes.

## Problem 38

* **Concept:** Smallest Positive Even Number
* **Time Complexity:** O(n)
* **Space Complexity:** O(1)
* **Lesson Learned:** Minimum tracking can be combined with multiple filters while using `None` as the initial value instead of `math.inf`.

## Problem 39

* **Concept:** Count Numbers Greater Than the Average
* **Time Complexity:** O(n)
* **Space Complexity:** O(1)
* **Lesson Learned:** Some problems naturally require multiple traversals. First compute the average, then perform a second traversal to count qualifying elements. Always handle the empty list before division.

## Problem 40

* **Concept:** Largest Negative Even Number
* **Time Complexity:** O(n)
* **Space Complexity:** O(1)
* **Lesson Learned:** Maximum tracking can be combined with multiple filtering conditions without changing the overall algorithm.

## Bonus Learning

* **Concept:** Combining Patterns
* **Lesson Learned:** Most problems are not completely new. They are combinations of familiar patterns such as counting, summation, tracking, first occurrence, last occurrence, and filtering. Identifying the underlying pattern before writing code makes problem-solving faster and more consistent.
